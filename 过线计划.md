# ModelNet40 PCT 过线计划

## Summary

目标是基于当前 `pct.py` 完成训练、验证、推理和 `result.json` 生成，提交格式满足 `prob.md`，并尽量稳定达到测试 Accuracy `>= 0.80`。不需要高性能工作站；RTX 3050Ti 4GB 可以做这题，但必须先让 Jittor 识别 CUDA。

当前环境中 `nvidia-smi` 能看到显卡，但 Jittor 报 `No CUDA found`，且系统里没有 `nvcc`，所以第一步是补好 CUDA toolkit/Jittor CUDA 环境。

## Key Changes

- 修改 `pct.py` 的运行入口：去掉强制 `jt.flags.use_cuda = 0`，新增 `--cuda {auto,on,off}`，默认 `auto`，启动时做 CUDA 可用性检查；不可用时只允许 CPU smoke test，不建议正式训练。
- 增加训练/推理 CLI：`--mode train,predict,train_predict`、`--save_path pct_best.pkl`、`--load_path pct_best.pkl`、`--result_path result.json`、`--val_ratio 0.1`、`--vote_num 10`。
- 增加 deterministic stratified train/val split：从 `train_labels.npy` 每类固定抽约 10% 做验证集，用 seed 固定。
- 保持 PCT 主结构不大改，优先做低风险提分：默认 `n_points=1024`、`batch_size=8`、`epochs=250`、`lr=0.001`、`weight_decay=1e-4`、cosine annealing、label smoothing 默认 `0.2`，数据增强保留旋转、缩放、平移、jitter，dropout 上限改为 `0.5`。
- 推理阶段使用 test-time voting：每个测试样本重复随机采样/预测 `vote_num=10` 次，平均 logits 后取类别；输出 JSON key 为 `"0"` 到 `"2467"`，value 为 `0-39` 的 int。

## Test Plan

- 数据检查：确认 `train_points.npy` shape 为 `(9843, 2048, 3)`，`test_points.npy` shape 为 `(2468, 2048, 3)`，类别数为 40。
- 环境检查：运行 Jittor CUDA 小算子；若仍报 `No CUDA found`，先不进入正式训练。
- 功能 smoke test：用小点数、小 batch、1 epoch 跑通训练、保存 checkpoint、加载 checkpoint、生成 JSON。
- JSON 校验：确认 `result.json` 有 2468 个 key，key 连续，所有预测为 `0-39` 整数。
- 训练验收：验证集 accuracy 稳定接近或超过 `80%` 后再生成提交；若验证集低于 `78%`，优先延长到 300 epochs 或把 `vote_num` 提到 20。

## Hardware Assumptions

- 3050Ti 4GB 足够完成热身赛，不需要高性能计算机。
- 推荐参数是为 4GB 显存保守设置的：`n_points=1024`、`batch_size=8`、`vote_num=10`。
- 不建议直接用 `n_points=2048` 加大 batch，因为 PCT self-attention 会产生 `N x N` 注意力矩阵，显存会快速上涨。
- CPU 可以做代码验证，但正式训练会很慢；若 CUDA 环境短时间修不好，才考虑借用云 GPU 或 Colab。
